Running Topology Optimization simulations

Introduction

In these tutorial files, you will learn how to use the various modules of Swan in order to generate your own FEM and Topology Optimization simulations. In each tutorial, a single building block will be analyzed, with some examples to make it easier. If you have any questions, feel free to create a new issue in the GitHub repository. In this tutorial, we will learn how to create and run a topology optimization simulation.

Macro

In order to run a topology optimization problem, the settings file needs to be generated. Some useful examples are test_cantilever.m, test_cantilever2.m, and test_cantilever3.m. We recommend that you copy the file that best suits your needs and customize it as needed.

List of parameters:

  • filename: contains the file with the mesh information.
  • ptype: MACRO/MICRO
  • method: material interpolation used (SIMPALL/SIMP_Adaptative/SIMP_P3)
  • materialType: always set to ISOTROPIC
  • initial_case: initial perturbation in the mesh for level-set cases (see LevelSetFactory.m)
  • cost: shape functionals to be included in the cost function
  • weights: weight of each of the shape functionals
  • contraint: constraints to be added to the optimization problem
  • constraint_case: type of constraint (EQUALITY/INEQUALITY)
  • optimizerUnconstrained: HAMILTON-JACOBI / SLERP / PROJECTED GRADIENT
  • optimizer: AlternatingPrimalDual / DualNestedInPrimal
  • incrementFactor
  • designVariable: Density / LevelSet
  • filterType: P1 / PDE
  • line_search_initiator

Initial and final volume fractions:

  • nsteps: steps between the initial Vfrac and the final Vfrac
  • Vfrac_initial: initial fraction of volume
  • optimality_initial
  • constr_initial
  • Perimeter_target
  • Vfrac_final: desidred final volume fraction
  • optimality_final
  • constr_final

Tolerance-related parameters:

  • TOL.rho_plus
  • TOL.rho_minus
  • TOL.E_plus
  • TOL.E_minus
  • TOL.nu_plus
  • TOL.nu_minus

Monitoring of the results:

  • plotting: can be set to true for 2D cases; will display the solution as the simulation progresses
  • printing: will export the results in the Output folder, can be visualized both in GiD and Paraview
  • printing_physics
  • monitoring: will display relevant information regarding the simulation (eg. cost, constraint and volume evolution)
  • monitoring_interval: iterations between updates to the monitoring
  • maxiter: maximum number of iterations
